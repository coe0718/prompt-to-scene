# Archiview Repo Auditor — Build Plan

## Pitch
> "Point Archiview at any GitHub repo. It reads the architecture, finds the bugs, scores the quality, and generates a fix plan. 10,000 lines analyzed in 30 seconds."

## What It Does
A web endpoint + CLI that takes a GitHub repo URL and produces:
- Architecture overview (key files, patterns, design decisions)
- Code quality score (0-100) across 5 dimensions
- Security findings (hardcoded secrets, injection risks, dependency issues)
- Documentation completeness audit
- Prioritized fix recommendations
- Downloadable HTML report (self-contained, offline)

## Target users
Open-source maintainers, team leads, anyone evaluating a codebase before contributing

## New builds

### 1. Repo Fetcher
- Accepts GitHub URL: `github.com/user/repo` or full `https://...`
- Uses GitHub API (no git clone needed — lighter, faster)
- Reads: repo metadata, directory tree, key files (README, package.json, Cargo.toml, etc.)
- Rate-limited: fetches top ~50 files by importance
- **Time: 3-4 hours**

### 2. Analysis Engine
- Chunks source files into batches for LLM
- Each chunk analyzed for: architecture role, code quality, security issues
- Multi-pass: first pass = structural analysis, second pass = deep analysis
- Aggregates scores across all files
- **Time: 4-5 hours**

### 3. Report Generator
- Beautiful HTML report with:
  - Header: repo name, stars, language, last commit
  - Score dashboard: 5 dimension scores (radar chart via canvas or SVG)
  - Architecture section: key files, data flow, patterns
  - Findings: categorized by severity (CRITICAL, WARNING, INFO)
  - Recommendations: prioritized, actionable
  - Footer: generated by Archiview, timestamp
- Self-contained, dark theme, matches landing page aesthetic
- **Time: 3-4 hours**

### 4. CLI Mode (stretch)
- `npx archiview-audit <repo-url>` 
- Outputs HTML report file
- **Time: 2 hours (after web version works)**
