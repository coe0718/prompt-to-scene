Architecture

Overview

Hermes Agent is a self-improving Python AI agent framework featuring a plugin-based architecture, multi-platform gateway supporting Telegram/Discord/CLI/etc., and a closed learning loop with autonomous skill creation and persistent memory. It unifies access to 200+ LLM providers and can spawn isolated subagents for parallel execution. The codebase is organized into modular plugins, deployment environments, optional skills, and packaging configurations.

WARNING security README.md
Node server lacks TLS - LAN-only deployment required
The node server uses bearer-token auth without TLS termination ('no TLS termination built in'). The README instructs users to 'run it on a LAN or behind a reverse proxy you trust.' This creates risk if users misconfigure networks or use untrusted proxies. Bearer tokens sent over plaintext WebSocket (ws://) are vulnerable to interception.
💡 Consider adding TLS support directly to the node server, or document more explicitly that WSS (wss://) must be used and that the bearer token should be treated as a high-privilege secret.
