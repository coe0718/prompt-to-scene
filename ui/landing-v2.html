How It Works
Three passes. One verdict.
PASS 01🔭

Structural Analysis

Reads README, config files, directory tree, build system. Maps the architecture, framework, and language before diving into code.

Model: Kimi K2.6
PASS 02🔬

Deep Code Review

Every source file chunked and analyzed for code quality, security vulnerabilities, maintainability, and anti-patterns. One file at a time.

Model: MiniMax M2.7
PASS 03🧠

Aggregation

All findings merged into prioritized scores across 5 axes. Generates the final report with radar chart, badge, and optional fix PR.

Model: Kimi K2.6
