How It Works
Three passes, one verdict
1

Structural Analysis

Hermes reads the README, config files, and directory tree to understand architecture, framework, language, and build system.

2

Deep Code Review

Source files are chunked and analyzed for code quality, security vulnerabilities, and maintainability — one file at a time.

3

Aggregate & Report

All findings are merged into a final report with scores, prioritized recommendations, risks, and a verdict.

Capabilities
What Hermes looks for
🏗️

Architecture Analysis

Framework detection, directory structure, design patterns, entry points, data flow — Hermes maps the entire codebase topology.

✅

Code Quality Scoring

Five dimensions scored 0-100: architecture, code quality, security, documentation, maintainability. Color-coded and ranked.

🔒

Security Auditing

Hardcoded secrets, injection risks, dependency vulnerabilities, unsafe patterns — flagged with severity levels and fix suggestions.

📋

Actionable Recommendations

Prioritized fix list with effort estimates and impact ratings. From "fix this today" to "consider for next sprint."

📊

Self-Contained Reports

Beautiful dark-themed HTML with all CSS inlined. Downloadable, shareable, printable. No server needed to view.

🔄

PR Generation

Critical findings can be turned into pull requests automatically. Hermes writes the code, you review and merge.
